# B3 review - Frontend backend connection

VERDICT: APPROVE

## Scope reviewed

- Disk selection now loads cheap scan roots in Tauri mode instead of scanning every root on startup.
- Browser/Vite mode still uses mock disks and keeps the visual city flow available.
- Selecting a live root performs one cached `scan_root` call and renders the returned bounded backend tree.
- Hydration preserves backend `path` values on `FsNode` for Explorer actions.
- Details panel routes live `Open in Explorer` actions through the Tauri command while retaining the friendly browser fallback.

## Performance review

- Startup avoids eager live scans and only calls `list_scan_roots` in Tauri mode.
- Live scan promises are cached per root path during the session.
- Tauri `invoke` is dynamically imported only when the runtime is available, keeping browser preview lean.
- No frontend filesystem traversal was added; map data still comes pre-aggregated from Rust.
