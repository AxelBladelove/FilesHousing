## Context

You are working in a Codex project worktree for FilesHousing after B2 has implemented real backend scanning. The UI currently still imports mock disks synchronously from `src/mockData.ts` in `src/main.ts`; `src/backend.ts` exists but is only partially wired. Your slice is to connect that ready UI to Tauri commands while keeping browser/Vite development usable with mock fallback.

Current important context:
- `src/backend.ts` already exposes `listScanRoots`, `scanRoot`, `openInExplorer`, `hasTauriBackend`, and `loadInitialDisks`, but it eagerly scans every root and is not used by the main UI.
- `src/types.ts` has `BackendFsNode.path`, but `hydrateNode` currently drops it from `FsNode`; real Explorer actions need the backend path preserved.
- `src/main.ts` creates `const disks = buildDisks()` at module load and disk cards always use that mock array.
- B2 scanner returns already-bounded `Disk`/`FsNode` trees; the frontend must not traverse the live filesystem.

## Requirements

1. Finish the frontend backend adapter in `src/backend.ts`: use Tauri `invoke` only when running inside Tauri and otherwise return mock data. Avoid scanning all roots during initial disk selection.
2. Update disk selection to list backend scan roots/drives when available, with mock disks in browser/Vite mode and if backend listing fails.
3. Update map entry flow so selecting a disk/root triggers exactly one real scan for that selected root, then feeds the existing city renderer. Cache the scanned result per root/path during the session so returning to disk select does not rescan repeatedly.
4. Keep existing scan overlay semantics, but make progress copy honest if the backend only returns a completed scan for now; do not imply per-file live progress before B2 exposes streaming progress.
5. Preserve all existing interactions: map zoom, search, filters, details, queue.
6. Preserve backend `path` on hydrated `FsNode`s and wire "Open in Explorer" through the backend command when live, with the existing friendly fallback note in browser mode.
7. Avoid raw filesystem traversal from the frontend; all scan data must arrive pre-processed from Rust.
8. Commit your work on your worktree branch with clear messages.
