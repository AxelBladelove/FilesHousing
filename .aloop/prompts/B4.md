## Context

You are working in a Codex project worktree for FilesHousing after B2 implemented real scanner output with paths and B3 connected the frontend to live backend scans with mock fallback. The UI already has a cleanup queue and details panel. `src/backend.ts` exposes `openInExplorer`, and live details buttons call the Tauri command `open_path_in_explorer` when a hydrated node has `path`. Your slice is safe backend action plumbing and validation, not aggressive cleanup automation or UI redesign.

Current important context:
- `src-tauri/src/commands.rs` has `open_path_in_explorer(path: String)` using `canonicalize()` and `explorer`.
- `src-tauri/src/models.rs` serializes absolute backend paths on `FsNode`.
- `src/types.ts` preserves `FsNode.path`, and `src/main.ts` uses it for the Explorer button in Tauri mode.
- The queue panel is still preview-only text; no backend cleanup/recycle command exists yet.

## Requirements

1. Harden backend path/action validation for commands that operate on paths:
   - reject empty paths, synthetic `#rest` paths, and non-existing targets;
   - avoid panics and return user-safe error strings;
   - keep symlink/reparse behavior conservative.
2. Improve `open_path_in_explorer` so Windows opens Explorer on the selected file/folder path when available, with a safe non-Windows fallback. Do not rename the command unless the frontend is updated too.
3. Add preview-style cleanup queue backend support if lightweight, e.g. a Tauri command that accepts explicit path + size estimates and returns a review summary/reclaimable bytes without deleting anything.
4. Do not implement destructive delete unless it is completely inert from the current UI and requires an explicit future confirmation path. Prefer leaving deletion as a future command design note over adding risky behavior.
5. Update frontend action calls only where necessary to use the preview command for the queue panel; no UI redesign.
6. Add focused Rust tests for validation/preview helpers where practical.
7. Commit your work on your worktree branch with clear messages.
