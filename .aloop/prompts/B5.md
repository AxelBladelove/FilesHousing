## Requirements

1. Extend Rust models and matching TypeScript backend types with:
   - `Actionability`: `cleanupCandidate`, `managedBundle`, `protectedInternal`, `systemProtected`, `informationalOnly`
   - `BundleKind`: `app`, `game`, `system`, `project`, `library`, `unknown`
   - `PreferredAction`: `review`, `reveal`, `openFolder`, `uninstall`, `manageBundle`, `none`
   - Node fields: `actionability`, `actionReason`, `bundleKind`, `preferredAction`, `isInternal`, `parentBundleName`
   - Serialize Rust fields as camelCase JSON and keep existing frontend hydration working.

2. Implement a fast heuristic semantic classifier with no heavyweight dependencies.
   - System protected:
     - paths under/matching `C:\Windows`, `System Volume Information`, `$Recycle.Bin`
     - `pagefile.sys`, `hiberfil.sys`, `swapfile.sys`
     - obvious `.sys` driver/system files under system paths
     - preferred action `none`
     - reason `System protected. Do not clean individually.`
   - Managed bundles:
     - directories that look like installed apps or games under components such as `Program Files`, `Program Files (x86)`, `WindowsApps`, `steamapps\common`, `XboxGames`, `Epic Games`, `GOG Games`, `Ubisoft`, `Riot Games`, `Battle.net`, `EA Games`, `Rockstar Games`
     - directories with local uninstall hints: `uninstall.exe`, `unins*.exe`, `uninstaller.exe`, obvious uninstall `.msi`
     - app/game root folders should be `managedBundle`, `bundleKind` app or game, `preferredAction` manageBundle, `isInternal=false`, reason `Managed app/game bundle. Manage or uninstall as a whole.`
   - Protected internals:
     - files/subfolders inside a managed app/game bundle should normally be `protectedInternal`, `isInternal=true`, `parentBundleName=<detected bundle name>`, preferred action `none` or `reveal`, reason `Internal app/game file. Manage the parent bundle instead.`
     - examples include `.pak`, `.dll`, `.bin`, `.dat`, `.ucas`, `.utoc`, `.sig`, shader/cache/asset files inside app/game folders
     - sizes must still count toward parent totals.
   - Cleanup candidates:
     - likely-safe individual review targets outside app/system folders, including large videos, archives, installers in Downloads/Desktop/user folders, backups/exports/dumps, recognizable user-controlled cache folders
     - preferred action `review`
     - reason should be short and specific, e.g. `Large video file.`, `Archive file.`, `Installer outside app folder.`, `Old backup/export.`
   - Informational only:
     - unknown large folders, mixed project folders, code repositories, ambiguous data folders
     - preferred action `openFolder` or `reveal`.

3. Scanner architecture:
   - Add a lightweight semantic context through scan recursion: inside system area, inside managed bundle, parent bundle name, parent bundle kind.
   - Detect whether current directory starts a bundle during directory scan.
   - Propagate bundle context to children.
   - Classify each node once as it is created.
   - For directories, allow final classification after child scan when child names reveal uninstallers or launcher metadata.
   - Avoid repeated expensive path scans and avoid recursively rescanning children solely for classification.

4. Fix Windows Explorer argument construction:
   - If target is a directory, open that directory directly.
   - If target is a file, open Explorer selecting the file.
   - Keep validation for empty paths, missing paths, synthetic `#rest`, symlinks/reparse points, unsupported entries.
   - Return clear error strings.
   - Factor command argument construction into a pure function if needed so tests are cross-platform.

5. Update `preview_cleanup` safety:
   - Reject `systemProtected`, `protectedInternal`, synthetic `#rest`.
   - Reject paths inside managed bundles unless the exact node is a top-level managed bundle and future preferred action is `manageBundle`.
   - Because frontend passes only paths and estimated bytes, add backend-side validation/path classifier so preview independently rejects dangerous paths.
   - Do not delete files. Preview only.

6. TypeScript compatibility:
   - Update `src/types.ts` only as needed so frontend receives new fields.
   - Hydrate new fields onto `FsNode`; do not implement UI changes.

7. Tests:
   - Add Rust tests for system path classification, Steam/Xbox/Epic/GOG-style game bundle classification, internal game asset classification, installer outside app folder as cleanup candidate, Explorer argument construction, and cleanup preview rejection of protected internals.
   - Use synthetic paths and temp dirs; do not depend on installed games.
